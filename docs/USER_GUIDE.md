# Remote Claude 飞书机器人用户手册

## 这是什么？

Remote Claude 是一个飞书机器人，让你可以**在飞书中操作电脑上运行的 Claude Code / Codex 进程**。

简单来说：电脑上用终端跑 Claude Code 写代码，同时在**手机飞书**上看进度、发指令、点按钮 — 不用守在电脑前，随时随地掌控 AI 编程。

### 核心场景

- **开会时**：Claude 在电脑上跑着任务，飞书弹出权限确认，手机上一键批准
- **通勤时**：手机飞书查看 Claude 的编码进度，发消息让它继续下一步
- **午休时**：躺在沙发上用飞书给 Claude 下指令，和在终端里操作一样
- **多人协作**：创建飞书群，多个人共同操作同一个 Claude 会话

### 飞书端能做什么？

- 查看 Claude 的实时输出（彩色代码、流式更新）
- 发送消息给 Claude（和终端里打字一样）
- 点击按钮选择选项、批准权限确认
- 查看后台 agent 运行状态
- 启动新的 Claude/Codex 会话
- 管理多个会话（切换、终止）

## 工作原理

理解一个关键概念：**飞书机器人本身不运行 Claude，Claude 进程跑在你的电脑上。**

```
你的电脑（macOS / Linux）
┌──────────────────────────────────┐
│  Claude Code 进程（终端中运行）     │
│       ↕                          │
│  Remote Claude（桥接服务）         │
│       ↕                          │
│  飞书长连接（WebSocket）           │
└──────────┬───────────────────────┘
           │ 互联网
           ↓
┌──────────────────────────────────┐
│  飞书服务器                       │
│       ↕                          │
│  飞书 App（手机/电脑）             │
└──────────────────────────────────┘
```

这意味着：
- **每个使用者**都需要在自己的电脑上安装并运行 Remote Claude
- 飞书机器人是共享的"入口"，但它连接的是**你自己电脑上的 Claude 进程**
- 你的电脑需要保持开机和联网

## 角色说明

使用这个机器人涉及两个角色：

| 角色 | 做什么 | 做一次还是每次？ |
|------|--------|-----------------|
| **管理员**（创建飞书应用的人） | 在飞书开放平台创建应用、配置权限、发布上线 | 只需做一次，之后同组织所有人共用这个机器人 |
| **使用者**（每个想用的人） | 在自己电脑上安装 Remote Claude、启动 Claude 会话 | 每台电脑安装一次，每次使用时启动 |

> 管理员和使用者可以是同一个人。如果你是第一个配置的人，两部分都要做。

## Part A：管理员配置（只需一次）

> 如果你的组织中已经有人创建好了飞书机器人，跳到 [Part B](#part-b使用者配置每台电脑一次)。

### A1. 创建飞书应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)，点击「创建企业自建应用」
2. 填写应用名称（如 "Remote Claude"）和描述
3. 在应用详情页记下 **App ID** 和 **App Secret**（后面每个使用者都需要用到）

### A2. 添加机器人能力

在应用页面：点击「添加应用能力」→ 选择「机器人」

### A3. 配置事件回调（长连接模式）

- `事件与回调` → `事件配置` → `订阅方式` 右边的笔图标 → 选择「使用长连接接收事件」→ 保存
- 下方「添加事件」→ 搜索添加：**接收消息 v2.0** (`im.message.receive_v1`)
- `事件与回调` → `回调配置` → `订阅方式` 右边的笔图标 → 选择「使用长连接接收回调」→ 保存
- 下方「添加回调」→ 搜索添加：**卡片回传交互** (`card.action.trigger`)

### A4. 配置权限

`权限管理` → `批量导入/导出权限` → 导入以下内容：

```json
{
  "scopes": {
    "tenant": [
      "cardkit:card:write",
      "contact:contact.base:readonly",
      "contact:user.id:readonly",
      "im:chat.members:read",
      "im:chat.members:write_only",
      "im:chat:create",
      "im:chat:read",
      "im:chat:update",
      "im:message.group_at_msg:readonly",
      "im:message.group_msg",
      "im:message.p2p_msg:readonly",
      "im:message.urgent",
      "im:message:readonly",
      "im:message:send_as_bot",
      "im:message:update",
      "im:resource"
    ]
  }
}
```

> 以上是机器人核心功能所需的最小权限集。完整权限列表见项目 [README](https://github.com/ShellyDeng08/lark-remote-claude)。

### A5. 发布应用

点击「创建版本」→ 填写版本说明 → 「发布到线上」→ 管理员审批通过

发布后，**把 App ID 和 App Secret 分享给需要使用的同事**（他们配置时要用）。

## Part B：使用者配置（每台电脑一次）

> 每个想通过飞书操作 Claude 的人都需要完成以下步骤。

### B1. 安装 Remote Claude

你的电脑需要是 **macOS 或 Linux** 系统。两种安装方式任选其一：

**方式 A：npm 安装**

```bash
npm install lark-remote-claude
```

**方式 B：源码安装**

```bash
git clone https://github.com/ShellyDeng08/lark-remote-claude.git
cd lark-remote-claude
./init.sh
```

安装完成后**重启终端**生效。

### B2. 配置飞书凭证

首次运行时，在终端执行 `cla`（或 `cl`），按照交互提示填入管理员提供的 **App ID** 和 **App Secret**。

也可以手动编辑 `~/.remote-claude/.env`：

```bash
FEISHU_APP_ID=cli_xxxxx      # 管理员提供
FEISHU_APP_SECRET=xxxxx      # 管理员提供
```

> 同一个飞书应用的 App ID / Secret 所有使用者共用，不需要每人创建应用。

### B3. 启动 Claude 会话

```bash
# 启动 Claude（以当前目录路径为会话名，同时自动启动飞书客户端）
cla

# 或者跳过权限确认模式
cl

# 启动 Codex
cx
```

启动成功后，你的 Claude 进程就可以通过飞书操作了。

### B4. 在飞书中找到机器人

1. 打开飞书（手机或电脑端），搜索管理员创建的机器人名称（如 "Remote Claude"）
2. 点击进入对话，发送 `/menu`
3. 菜单卡片会显示你电脑上正在运行的 Claude 会话，点击即可连接

> 第一次搜索可能比较慢。如果搜不到，确认应用已发布上线。

## 日常使用

完成上面的配置后，日常使用流程很简单：

### 每次使用

1. **电脑终端**：运行 `cl`（或 `cla` / `cx`）启动 Claude 会话
2. **飞书**：给机器人发 `/menu`，点击会话名连接
3. 开始对话 — 直接发消息，和在终端里打字一样

### 命令速查

| 命令 | 说明 |
|------|------|
| `/menu` | 打开快捷操作面板（推荐入口） |
| `/list` | 列出所有可用会话（带一键连接按钮） |
| `/attach <会话名>` | 连接到指定会话 |
| `/detach` | 断开当前连接 |
| `/start <会话名> [工作路径]` | 从飞书端启动新的 Claude 会话 |
| `/kill <会话名>` | 终止某个会话 |
| `/status` | 查看当前连接状态 |
| `/ls [路径]` | 查看文件列表 |
| `/tree [路径]` | 查看目录树 |
| `/new-group <会话名>` | 创建专属群聊（多人协作） |
| `/help` | 显示帮助 |

### 和 Claude 对话

连接会话后，**直接发消息**即可：

```
帮我重构这个函数，使用 async/await
```

Claude 的回复会以**流式卡片**实时显示在飞书中，包括彩色代码高亮。

### 处理交互（选项 / 权限确认）

当 Claude 需要你确认时，卡片上会自动出现按钮：

- **权限确认**：卡片 header 变红 🔐，点击「Yes」或「No」
- **选项选择**：卡片 header 变蓝 🤔，点击对应选项按钮

### 从飞书端启动新会话

即使不在电脑前，也可以从飞书端启动新的 Claude 会话：

```
/start mywork ~/dev/myproject
```

前提是你的电脑上 Remote Claude 的飞书客户端进程在运行（`cla` / `cl` 启动时会自动运行）。

### 多人协作

创建专属群聊，群里所有人共同操作同一个 Claude：

```
/new-group mywork
```

## 卡片界面说明

### Header 状态颜色

| 颜色 | 含义 |
|------|------|
| 🟠 橙色 | Claude 正在思考/执行中 |
| 🔴 红色 | 等待权限确认 |
| 🔵 蓝色 | 等待你选择选项 |
| 🟢 绿色 | Claude 就绪，等待输入 |
| ⚪ 灰色 | 连接已断开 / 历史记录 |

### 卡片结构

- **内容区**：Claude 的输出（你的消息 ❯ 开头、Claude 回复、工具执行结果）
- **状态区**：当前执行状态、后台 agent 信息
- **按钮区**：选项/权限确认按钮（有交互时出现）
- **底部**：菜单、断开/重连、Enter ↵

### 流式更新

Claude 边思考边输出时，卡片实时更新。内容较长时，当前卡片会自动冻结（变灰），并开一张新卡片继续显示。

## 高级功能

### 菜单内的开关

`/menu` 卡片中有几个开关：

- **就绪通知**：Claude 完成任务时，飞书发送提醒
- **加急通知**：以加急消息提醒（更醒目）
- **新会话跳过确认**：新启动的会话自动 bypass permissions

### 用户白名单

在 `~/.remote-claude/.env` 中限制谁能使用机器人：

```bash
ENABLE_USER_WHITELIST=true
ALLOWED_USERS=ou_xxxxx,ou_yyyyy
```

### 自定义 CLI 命令

如果 Claude CLI 命令不是默认的 `claude`：

```bash
# ~/.remote-claude/.env
CLAUDE_COMMAND=ccr code
```

### 快捷命令一览

| 终端命令 | 说明 |
|---------|------|
| `cla` | 启动 Claude（需确认权限） |
| `cl` | 启动 Claude（跳过权限确认） |
| `cx` | 启动 Codex（跳过权限确认） |
| `cdx` | 启动 Codex（需确认权限） |
| `remote-claude list` | 查看所有会话 |
| `remote-claude attach <名称>` | 从另一个终端连接会话 |
| `remote-claude lark status` | 查看飞书客户端运行状态 |

## 常见问题

### 搜索不到机器人？
- 确认飞书应用已「创建版本」并「发布到线上」
- 第一次搜索有延迟，等几分钟再试
- 确认你和机器人在同一个飞书组织下

### 发消息没有反应？
- 确认你的电脑上飞书客户端进程在运行：终端执行 `remote-claude lark status`
- 如果未运行，执行 `remote-claude lark start` 或重新运行 `cla` / `cl`
- 检查是否启用了白名单但你不在名单中

### `/list` 显示没有会话？
- 你需要先在电脑终端运行 `cla` / `cl` / `cx` 启动一个 Claude 会话
- 飞书机器人只能看到**你自己电脑上**正在运行的会话

### 连接会话后看不到内容？
- 发送 `/list` 确认会话存在
- 尝试 `/detach` 再 `/attach` 重新连接

### 卡片按钮点击没反应？
- 确认飞书开放平台已配置「卡片回传交互」回调
- 确认电脑上飞书客户端进程正常运行

### 换了一台电脑怎么办？
- 在新电脑上重新执行 [Part B](#part-b使用者配置每台电脑一次) 的步骤即可
- 飞书端不需要重新配置，机器人是同一个

### 能否在非企业飞书中使用？
- 目前仅支持飞书企业版（需要企业自建应用能力）

### 支持 Codex 吗？
- 支持。使用 `cx` 或 `cdx` 启动 Codex 会话，飞书端操作方式完全一样

---

## 安全说明

- **不侵入 Claude 进程**：所有远程功能通过终端交互（PTY）实现，Claude 本身不受影响
- **无需公网端口**：通信链路为本地 Unix Socket + 飞书长连接（WebSocket）
- **建议启用白名单**：限制可操作的用户范围，避免未授权访问
- **操作可审计**：所有操作日志在 `~/.remote-claude/lark_client.log`
